05 — Predicted Questions

What may come in your exam

These are not random — each one is based on the rotation pattern of the last 3 years. Probability ratings are my best estimate based on the organizer's coverage and PYQ rotation.

Group C — Long Answer (likely, 15 marks) VERY HIGH

Distance Vector Routing with example + count-to-infinity problem. (Direct hit 2024-25, indirect in 2023-24.)

How to answer: Draw a 3-4 node graph, show distance vectors updating step-by-step using Bellman-Ford. Then explain count-to-infinity with a 3-node example where a link drops. Mention split horizon + poison reverse as solutions. 5-mark subpart: show the routing table at each iteration.
Group C — Long Answer (likely, 15 marks) VERY HIGH

Subnetting problem — likely a multi-part numerical: ISP address block design (like 2024-25 Q11a) or first/last address calculation (like 2023-24 Q5, 2022-23 Q8c).

How to answer: Show the binary of the last octet, apply the prefix mask, then find block size. For multi-group ISP: arrange from largest to smallest group. Tip: Practice 190.100.0.0/16 style problems 2-3 times — they're formulaic once you get it.
Group C — Long Answer (very likely, 15 marks) VERY HIGH

Short notes on 3-4 application-layer protocols — DNS, FTP, SMTP, Telnet, HTTP. (Direct hit 2024-25 Q10.)

How to answer: For each protocol — port number, key function, one diagram. DNS: resolver + root/TLD/auth servers. FTP: control (21) + data (20) channels. SMTP: push protocol on port 25. Telnet: virtual terminal on port 23.
Group B — Short Answer (high, 5 marks) HIGH

Compare Circuit Switching vs Packet Switching — 2024-25 asked this directly. 2023-24 was a related switching question. Due for re-ask.

How to answer: Make a 5-row table — definition, resource allocation, delay, suitability, example. Mention datagram vs virtual circuit packet switching as the two flavours.
Group B — Short Answer (high, 5 marks) HIGH

Piggybacking — full 5-mark question with advantages, OR piggybacking as a VSA fill-in. Seen in 2023-24 and 2024-25.

How to answer: Definition (sender delays ACK to attach it to next outgoing data), 3 advantages (better bandwidth use, fewer frames, control overhead down), 1 disadvantage (delayed ACK if no data).
Group C — Long Answer (medium, 15 marks) MEDIUM

Leaky Bucket + Token Bucket combined question with diagrams. 2023-24 Q9c was exactly this. Examiner loves rephrasing the same question — fresh wording, same core.

How to answer: Two diagrams (bucket with hole at bottom for leaky; bucket with tokens added at constant rate for token), a 4-column comparison, and a 1-paragraph "how combining both gives best QoS" close.
Group C — Long Answer (medium, 15 marks) MEDIUM

TCP header + 3-way handshake as one 15-mark question. Or split as 7-mark header (with diagram) + 8-mark handshake.

How to answer: Draw the 20-byte header with all fields labelled (Source/Dest Port, Seq, Ack, Offset, Reserved, Flags URG/ACK/PSH/RST/SYN/FIN, Window, Checksum, Urgent, Options, Padding). For handshake: 3 time-stamped packets showing client→server SYN, server→client SYN+ACK, client→server ACK.
Group A — VSA (almost certain, 1 mark each) GUARANTEED

10 VSA fillers — DNS definition, port number of a known protocol, MAC address type, ARQ window size formula, classification by IP class (A/B/C/D/E), function of a layer.

Memorise: Port numbers (20, 21, 23, 25, 53, 80, 110), IPv4 class ranges (A:1-126, B:128-191, C:192-223, D:224-239, E:240-255), loopback (127.0.0.1), MAC size (48 bit), IPv4 size (32 bit).
06 — Chapter-Wise Roadmap

What to study, in what order

Based on the organizer's 8-chapter structure, weighted by how MAKAUT's exam actually distributes marks. Click each chapter to expand the must-study topics.

1 Overview of Data Communication & Networking ~8-10 marks
  • OSI 7-layer model — names, functions, PDU at each layer (bit/frame/packet/segment/data)
  • TCP/IP 4-layer model and comparison with OSI
  • Network topologies — Mesh, Star, Bus, Ring, Tree, Hybrid (with diagrams)
  • Transmission modes — Simplex, Half-duplex, Full-duplex
  • Multiplexing — FDM, TDM, WDM (basic idea)
  • Bandwidth vs Throughput definitions
2 Physical Layer & Data Link Layer ~12-15 marks
  • Guided media — Twisted Pair, Coaxial, Optical Fiber (advantages, disadvantages, comparison)
  • Line coding — NRZ-L, NRZ-I, Manchester, Differential Manchester (with diagrams)
  • Transmission impairments — Attenuation, Distortion, Noise
  • Data Link Layer functions — framing, error control, flow control, access control
  • Error detection & correction — VRC/LRC, CRC (with example), Hamming code (with example)
  • Bit stuffing — algorithm + worked example (5 consecutive 1s rule)
  • Repeater, Bridge, Router, Gateway, Hub, Switch — layer & function
3 Medium Access Sub Layer (MAC) ~12-15 marks
  • Random access — ALOHA (pure & slotted), CSMA, CSMA/CD, CSMA/CA — efficiency, vulnerable time, throughput formulas
  • Controlled access — Reservation, Polling, Token Passing
  • Channelization — FDMA, TDMA, CDMA (concept only)
  • Ethernet — frame format, MAC addressing, 48-bit address
  • MAC sublayer functions — addressing, channel access, collision resolution
  • IEEE 802.x standards (basic awareness)
4 Network Layer ~20-22 marks (HEAVIEST)
  • IPv4 addressing — classful (A/B/C/D/E), classless (CIDR with /n notation)
  • Subnetting — finding network address, first/last host, broadcast — practice 5+ numericals
  • Subnetting design — VLSM (Variable Length Subnet Masking) for ISP-style problems
  • Supernetting / CIDR — combining blocks
  • ARP, RARP, ICMP, DHCP — protocol functions
  • Routing algorithms — Distance Vector (Bellman-Ford), Link State (Dijkstra)
  • Routing protocols — RIP, OSPF, BGP (basics)
  • Count-to-infinity + Split Horizon + Poison Reverse (solutions)
  • Two-node instability problem in DVR (2023-24 asked this!)
  • IPv6 — header format, why needed (awareness only)
5 Transport Layer ~15-18 marks
  • UDP vs TCP — comparison, header format, when to use which
  • TCP header — all 20-byte fields (very important — full marks question)
  • 3-way handshake — SYN → SYN+ACK → ACK (with diagram)
  • Congestion control — Leaky bucket, Token bucket (definitive topic)
  • Open-loop vs Closed-loop congestion control techniques
  • ARQ protocols — Stop-and-Wait, Go-Back-N, Selective Repeat (with diagrams)
  • Piggybacking — definition + advantages + timing diagram
  • Port numbers — well-known, registered, dynamic ranges
  • Socket — IP + port pair concept
6 Application Layer & Modern Topics ~8-10 marks
  • DNS — namespace, resolver, root/TLD/auth servers, iterative vs recursive query
  • FTP — control channel (21), data channel (20), active vs passive mode
  • SMTP, POP3, IMAP — email flow
  • HTTP / WWW — request-response model
  • Telnet — virtual terminal (port 23)
  • Network security basics — Firewall (packet-filter, stateful, application), PGP, VPN
  • Symmetric vs Asymmetric encryption (with block diagram)
  • Bluetooth architecture — piconet, scatternet, protocol stack (awareness)

08 — Exam Hall Tactics

6 things that will save you marks

The paper is designed to be passed. These tips come from how MAKAUT actually grades — they reward structure and diagrams.

01
Always draw diagrams
For every protocol (3-way handshake, DVR, ALOHA, Token bucket) draw the diagram. Half the marks for many questions are for "neat labelled diagram".
02
Use comparison tables
For "compare X and Y" questions, a 4-5 column table gets you more marks than 3 paragraphs. Examiners scan tables — they see all points at once.
03
Number your steps in numericals
Subnetting, CRC, Hamming — show every step. Step 1: convert to binary. Step 2: apply mask. Don't skip. Even if the final answer is wrong, you get partial marks.
04
Skip VSA fillers you don't know
Group A is 1 mark each, you pick 10 of 12. Don't waste 5 minutes on a VSA — move on, come back later. Time is your enemy in 3-hour paper.
05
Pick best 3 of 5 in each group
You don't have to answer the first 3. Skim all 5 questions in Group B and C, then pick the 3 you know best. Many students answer the easiest 3, not the first 3.
06
Read 2023-24 Q9c carefully
It's a Leaky+Token combined question — if the same question (or close variation) appears, you'll be ready. Examiner reuses frameworks; they rarely invent new question types.

Quick Summary — the bottom line

If you read nothing else, read this. These are the highest-yield points for the next 3-4 hours of your study.

Top 5 Topics (must do)

1. Leaky & Token Bucket 2. Distance Vector Routing 3. TCP Header + 3-way Handshake 4. Subnetting numericals 5. Short notes on DNS/FTP/Firewall/Telnet

Where the marks hide

Group C (45 marks) decides your grade. Group A (10 marks) is for confidence + buffer. Don't spend 2 hours on VSA — that's a common mistake.

Easy 30 marks you should not lose

Group A (10) + 2 of Group B (10) are almost free if you revise the organizer once. The other 10 from Group B is moderate difficulty. That leaves 30 marks for the "hard" Long Answer questions — but you only need 3 of 5.

Last word of advice

Don't try to learn new things in the last 24 hours. Re-revise the things you already half-know. That's where marks come from. Sleep 7-8 hours the night before. All the best.
